# ترمينال التداول اللحظي — Crypto Day-Trading Terminal (MVP)

منصة ويب لمتابعة وتحليل العملات الرقمية لحظياً، مبنية على بيانات Binance مباشرة، مع محرك تحليل فني قائم على قواعد ثابتة (بدون AI)، ومحفظة تداول افتراضية، وتنبيهات داخل التطبيق.

هذا المشروع يغطي **نطاق الـ MVP المتفق عليه فقط**:

- ✅ شارت Candlestick لحظي (1m / 5m / 15m / 1H) مع EMA9 / EMA21 / MACD / Volume
- ✅ بيانات لحظية من Binance (REST للبيانات التاريخية + WebSocket للتحديث الحي)
- ✅ محرك إشارات Rule-Based (بدون AI): RSI، تقاطع EMA، ارتفاع الحجم → Trend + Signal (شراء/بيع/انتظار) + درجة ثقة
- ✅ محفظة افتراضية (Paper Trading) برصيد 10,000$، تسجيل صفقات يدوي أو من الإشارة مباشرة، حساب الأرباح/الخسائر، سجل الصفقات
- ✅ تنبيهات داخل التطبيق (صوت + Toast + سجل) عند ظهور إشارة شراء/بيع أو الاقتراب من دعم/مقاومة

العناصر التالية **مؤجَّلة عمداً لمراحل لاحقة** (موجودة في المستندات الأصلية لكنها خارج هذا الـ MVP): تداول حقيقي فعلي، تحليل AI توليدي، الأسهم الأمريكية، Auto-Trading Bot، Opportunity Scanner، Sessions Tracker، Fear & Greed، Whale Alerts، Trading Journal. البنية مصممة بحيث يسهل إضافتها لاحقاً دون إعادة هيكلة جذرية.

## ⚠️ تنويه مهم قبل التشغيل

تم بناء هذا الكود بالكامل **بدون اتصال إنترنت** في بيئة الإنشاء (sandbox)، لذلك لم أتمكن من تشغيل `npm install` أو تجربة المشروع فعلياً قبل تسليمه لك. الكود تمت مراجعته يدوياً بعناية (فحصت توازن الأقواس، وتطابق كل import مع الملف والدالة المصدّرة فعلياً)، لكن هناك احتمال وجود خطأ بسيط لم يظهر إلا عند التجميع الفعلي.

**إذا واجهت أي خطأ عند تشغيل `npm install` أو `npm run dev` أو `npm run build`:**
انسخ رسالة الخطأ كاملة وأرسلها لي هنا مباشرة، وسأصلحها لك فوراً.

## التشغيل

```bash
npm install
npm run dev
```

ثم افتح المتصفح على `http://localhost:5173`.

للتحقق من خلو الكود من أخطاء TypeScript قبل أي نشر فعلي:

```bash
npm run build
```

> ملاحظة: `npm run dev` (Vite) لا يفحص أخطاء الأنواع (Type Errors) أثناء التطوير — فقط `npm run build` يفعل ذلك عبر `tsc`. لذلك يُفضّل تشغيل `npm run build` مرة واحدة على الأقل للتأكد من خلو الكود من أي خطأ نوع قبل النشر.

## ملاحظة جغرافية مهمة

موقع `api.binance.com` و `stream.binance.com` محجوبان في بعض الدول (مثل الولايات المتحدة). إذا ظهرت لك أخطاء اتصال غريبة رغم أن الإنترنت يعمل، تأكد أن موقعك الجغرافي/الشبكة لا تحجب Binance. الحل البديل لاحقاً: التبديل إلى Binance.US أو تمرير الطلبات عبر خادم وسيط (Proxy/Edge Function) بدل الاتصال المباشر من المتصفح.

## هيكل المشروع

```
src/
  types/        تعريفات TypeScript للبيانات (سوق، تحليل، تداول، تنبيهات)
  services/     الاتصال بـ Binance (REST + WebSocket) — معزول تماماً عن React
  utils/        حسابات رياضية بحتة (EMA/RSI/MACD)، محرك الإشارات، منطق المحفظة، الصوت، التخزين
  context/      إدارة الحالة العامة (React Context): بيانات السوق، التنبيهات، المحفظة
  hooks/        useKlineSeries (شموع لحظية)، useSignalAlerts (تفعيل التنبيهات تلقائياً)
  components/   مكوّنات الواجهة، مقسّمة حسب القسم (watchlist, terminal, alerts, portfolio, layout, common)
  pages/        TerminalPage و PaperTradingPage — تجميع المكوّنات في صفحات كاملة
```

**مبدأ التصميم الأساسي:** كل بيانات المحفظة الافتراضية مشتقة من سجل الصفقات (`trades[]`) فقط كمصدر وحيد للحقيقة (`utils/portfolio.ts`) — لا يوجد رصيد أو مركز مخزّن بشكل منفصل قد يتعارض مع السجل. هذا يسهّل التوسع لاحقاً (مثل إضافة Win Rate أو Equity Curve) بدون تعديل البنية.

البيانات تُحفظ محلياً في المتصفح (`localStorage`): قائمة المراقبة وسجل الصفقات يبقيان بعد إعادة تحميل الصفحة. لا يوجد خادم خلفي (Backend) في هذا الـ MVP — كل شيء يعمل من المتصفح مباشرة.

## التوسع لاحقاً

البنية مقسّمة بشكل يسمح بإضافة هذه المراحل لاحقاً دون كسر الموجود:

- **الأسهم الأمريكية**: إضافة `services/` جديد بنفس نمط `binanceRest.ts`/`binanceSocket.ts` لمزود بيانات الأسهم، مع توحيد الواجهة عبر نفس أنواع `Kline`/`MiniTicker`.
- **طبقة تحليل AI**: يمكن إضافتها كخطوة إضافية بعد `analyzeKlines()` في `utils/signals.ts`، تستهلك نفس `AnalysisResult` وتضيف تفسيراً نصياً مولّداً فوقه.
- **Auto-Trading Bot**: يحتاج طبقة تنفيذ حقيقية تتصل بمفاتيح Binance API الخاصة بالمستخدم (Trade Execution API) — يُنصح ببناء هذه الطبقة كخدمة خلفية منفصلة (Backend) لأسباب أمنية (لا يجب تخزين مفاتيح API الحقيقية في كود الواجهة الأمامية مطلقاً).
- **Opportunity Scanner / Sessions Tracker / Fear & Greed / Whale Alerts**: كل منها يمكن أن يكون صفحة/مكوّن جديد مستقل يستهلك `MarketDataContext` الحالي أو يضيف Context جديد خاص به بنفس النمط.
